# Droidcon USA 2026 deck — how to run the slides

The talk deck is a [Slidev](https://sli.dev) presentation. Entry file:
`samdroidcon.md`. Package manager: **bun**.

## Quick start (deps already installed)

If `node_modules/` is present in this directory, just start the dev server:

```bash
cd droidcon
./node_modules/.bin/slidev samdroidcon.md --port 3030 --open
```

Then open:

- **Slides:** http://localhost:3030/
- **Presenter mode** (speaker notes + next-slide preview): http://localhost:3030/presenter/
- **Overview grid** (jump to any slide): http://localhost:3030/overview/

It's a hot-reload dev server — edits to `samdroidcon.md` refresh live in the
browser. Stop it with `Ctrl-C` (or `q` in the terminal running it).

If port 3030 is taken, pick another with `--port 3031`.

## First-time setup (fresh clone / no node_modules)

```bash
cd droidcon
bun install
```

Heads up — these three files are **gitignored**, so a fresh clone won't have
them and you'll need to recreate them before `bun install` will work:

- `.npmrc` — only needed if your environment defaults to an internal npm
  mirror that can't resolve these packages; all deps are public, so
  `registry=https://registry.npmjs.org/` works.
- `bun.lock` — lockfile (regenerated by `bun install`).
- `node_modules/` — installed deps.

Deps: `@slidev/cli`, `@slidev/theme-seriph`, `playwright-core` (see
`package.json`).

## Export to PDF / PNG (optional)

```bash
cd droidcon
./node_modules/.bin/slidev export samdroidcon.md         # -> slides-export.pdf
./node_modules/.bin/slidev export samdroidcon.md --format png
```

## Companion files

- `samdroidcon.md` — the deck (slides + speaker notes in `<!-- -->` blocks).
- `samdroidcon-notes.md` — long-form talk notes / rehearsal log.
- `samdroidcon-research.md`, `samdroidcon-repo-map.md` — background material.
- `public/` — images, video, QR assets referenced by the deck.
